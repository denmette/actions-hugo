## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Bring the action's runtime dependency set up to date with the approved GitHub Actions toolkit baseline from the planning task, and remove runtime code that is only present to support obsolete HTTP dependencies. Initial implementation showed that a direct switch to the Node built-in fetch is not compatible with the repository's current Jest 26 environment, and newer @actions/core dependencies introduce node:-specifier and undici-based assumptions that need to be accounted for explicitly. This task should update runtime packages in package.json/package-lock.json and implement the selected HTTP/version-fetching approach so the action continues to resolve Hugo versions and install binaries correctly.
