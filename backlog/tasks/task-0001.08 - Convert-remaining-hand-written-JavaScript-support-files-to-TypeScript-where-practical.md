## Implementation Notes

<!-- SECTION:NOTES:BEGIN -->
2026-04-08: After TASK-0001.06, the repository's only remaining hand-written JavaScript files are `jest.config.cjs` and `__tests__/node-compat/globals.cjs`. This task exists to decide whether those should be migrated to TypeScript or intentionally left in JavaScript/CommonJS due to toolchain constraints.

Execution note on April 8, 2026:
- `__tests__/node-compat/globals.cjs` is practical to convert to TypeScript because it is just repository-owned setup logic.
- `jest.config.cjs` should remain CommonJS unless the repository also adopts an additional config loader for Jest TypeScript configs. Converting that file blindly would add toolchain coupling without meaningful product value.

Completed on April 8, 2026 on branch `chore/task-0001-08-typescript-support-files`.

Outcome:
- Converted the hand-written Jest setup shim from `__tests__/node-compat/globals.cjs` to `__tests__/node-compat/globals.ts`.
- Updated `jest.config.cjs` to load the TypeScript setup file.
- Tightened `tsconfig.json` to compile only `src/**/*.ts`, so production `tsc` remains scoped to actual source files while test support files stay outside the production build boundary.
- Left `jest.config.cjs` as CommonJS intentionally. Under the current toolchain, converting the Jest config itself to TypeScript would require an additional config-loader path for Jest and would not improve runtime or product behavior enough to justify the added complexity.
