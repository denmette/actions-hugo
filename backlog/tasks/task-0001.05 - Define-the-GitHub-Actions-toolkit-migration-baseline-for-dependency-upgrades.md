## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Establish the compatibility baseline for upgrading the repository's @actions/* runtime dependencies before further implementation work proceeds. Current investigation found that replacing node-fetch with the Node built-in fetch breaks under the repository's current Jest 26 setup, and upgrading to newer @actions/core transitively introduces node:-specifier and undici-based runtime assumptions that the current test/tooling stack does not fully support. This task should determine the approved migration target for the GitHub Actions toolkit packages and document the required follow-on changes for runtime code, Jest, TypeScript, bundling, and action metadata.
<!-- SECTION:DESCRIPTION:END -->
