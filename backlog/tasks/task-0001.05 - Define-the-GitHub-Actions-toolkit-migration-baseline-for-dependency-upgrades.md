## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Establish the compatibility baseline for upgrading the repository's @actions/* runtime dependencies before further implementation work proceeds. Current investigation found that replacing node-fetch with the Node built-in fetch breaks under the repository's current Jest 26 setup, and upgrading to newer @actions/core transitively introduces node:-specifier and undici-based runtime assumptions that the current test/tooling stack does not fully support. This task should determine the approved migration target for the GitHub Actions toolkit packages and document the required follow-on changes for runtime code, Jest, TypeScript, bundling, and action metadata.
<!-- SECTION:DESCRIPTION:END -->

Approved conservative baseline for this initiative: stay on the CommonJS-compatible GitHub Actions toolkit line represented by @actions/core 1.x, @actions/exec 1.x, @actions/io 1.x, and @actions/tool-cache 2.x rather than upgrading to the newer ESM-oriented toolkit majors now.

Investigation showed that built-in fetch is not available in the current Jest environment and newer toolkit packages introduce node:-specifier and undici/http-client assumptions that should be handled in a separate migration.

Downstream implementation guidance: TASK-0001.01 should remove node-fetch v2-specific code using a CommonJS-compatible HTTP approach, preferably aligned with the toolkit HTTP client behavior, without requiring the newer ESM toolkit major line.

Future-state planning: TASK-0001.06 tracks the later migration to the newer ESM-oriented @actions/* toolkit majors after the conservative dependency cleanup is complete and stable.

Validation takeaway for downstream tasks: runtime/tooling tasks should treat integration-test environment assumptions and timeouts as a known repo constraint when verifying dependency changes.
